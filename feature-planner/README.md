# Feature Planner

TDD(Test-Driven Development) 기반의 단계별 기능 구현 계획을 생성하는 스킬입니다.

## 개요

복잡한 기능을 작고 관리 가능한 단계(phase)로 분해하여, 각 단계마다 테스트를 먼저 작성하고 구현하는 TDD 워크플로우를 따릅니다. 각 단계는 1-4시간 이내에 완료할 수 있도록 설계되며, 품질 게이트를 통과해야 다음 단계로 진행할 수 있습니다.

## 주요 기능

- **TDD 기반 워크플로우**: Red-Green-Refactor 사이클을 따르는 단계별 구현
- **품질 게이트**: 각 단계마다 테스트, 린트, 빌드 검증 필수
- **진행 상황 추적**: 마크다운 체크박스로 진행도 관리
- **리스크 평가**: 기술적, 의존성, 타임라인 리스크 사전 파악
- **롤백 전략**: 각 단계별 롤백 방법 문서화

## 사용 방법

### 1. 스킬 트리거

다음과 같은 요청 시 자동으로 실행됩니다:
- "이 기능을 단계별로 계획해줘"
- "TDD로 구현 계획 세워줘"
- "feature plan 작성해줘"
- 키워드: plan, planning, phases, breakdown, strategy, roadmap

### 2. 계획 생성 프로세스

1. **요구사항 분석**: 코드베이스 구조 파악, 의존성 확인
2. **단계 분해**: 3-7개의 단계로 기능 분해 (각 단계 1-4시간)
3. **계획 문서 생성**: `docs/plans/PLAN_<feature-name>.md` 생성
4. **사용자 승인**: 계획 실행 전 명시적 승인 요청
5. **문서 저장**: 체크박스 포함 계획 문서 저장

### 3. 각 단계 구조

모든 단계는 TDD 사이클을 따릅니다:

#### 🔴 RED: 실패하는 테스트 작성
- 구현 전에 테스트 먼저 작성
- 테스트가 실패하는지 확인

#### 🟢 GREEN: 최소 구현
- 테스트를 통과하는 최소한의 코드 작성
- 모든 테스트가 통과하는지 확인

#### 🔵 REFACTOR: 코드 개선
- 테스트를 유지하면서 코드 품질 향상
- 중복 제거, 네이밍 개선, 구조 최적화

### 4. 품질 게이트

각 단계 완료 시 다음 항목을 검증해야 합니다:

**TDD 준수**:
- [ ] Red 단계: 테스트 먼저 작성 및 실패 확인
- [ ] Green 단계: 구현 후 테스트 통과
- [ ] Refactor 단계: 리팩토링 후에도 테스트 통과
- [ ] 커버리지: 비즈니스 로직 ≥80%

**빌드 & 테스트**:
- [ ] 빌드 성공
- [ ] 모든 테스트 통과
- [ ] 테스트 실행 시간 적절

**코드 품질**:
- [ ] 린트 통과
- [ ] 포맷팅 일관성
- [ ] 타입 체크 통과

**보안 & 성능**:
- [ ] 보안 취약점 없음
- [ ] 성능 저하 없음
- [ ] 에러 처리 적절

## 계획 문서 구조

생성되는 계획 문서(`docs/plans/PLAN_<feature-name>.md`)는 다음을 포함합니다:

- 개요 및 목표
- 아키텍처 결정 사항
- 의존성 목록
- 테스트 전략
- 단계별 구현 계획 (체크박스 포함)
- 품질 게이트 체크리스트
- 리스크 평가
- 롤백 전략
- 진행 상황 추적
- 노트 & 학습 사항

## 단계 크기 가이드

**Small Scope** (2-3 단계, 3-6시간):
- 단일 컴포넌트 또는 간단한 기능
- 의존성 최소
- 예: 다크 모드 토글, 새 폼 컴포넌트

**Medium Scope** (4-5 단계, 8-15시간):
- 여러 컴포넌트 또는 중간 복잡도 기능
- 통합 복잡도 존재
- 예: 사용자 인증 시스템, 검색 기능

**Large Scope** (6-7 단계, 15-25시간):
- 복잡한 기능, 여러 영역에 걸침
- 아키텍처 영향 큰 변경
- 예: AI 기반 검색, 실시간 협업 기능

## 예시

```markdown
# 요청
"사용자 프로필 편집 기능을 TDD로 구현하고 싶어"

# 생성되는 계획 문서
docs/plans/PLAN_user-profile-edit.md

## Phase 1: Foundation
- 🔴 RED: 프로필 모델 테스트 작성
- 🟢 GREEN: 프로필 모델 구현
- 🔵 REFACTOR: 모델 코드 정리
- Quality Gate: 유닛 테스트 ≥80% 커버리지

## Phase 2: API Layer
- 🔴 RED: 프로필 업데이트 API 테스트
- 🟢 GREEN: API 엔드포인트 구현
- 🔵 REFACTOR: API 코드 정리
- Quality Gate: 통합 테스트 통과

## Phase 3: UI Components
- 🔴 RED: 프로필 편집 폼 테스트
- 🟢 GREEN: 폼 컴포넌트 구현
- 🔵 REFACTOR: 컴포넌트 구조 개선
- Quality Gate: E2E 테스트 통과
```
