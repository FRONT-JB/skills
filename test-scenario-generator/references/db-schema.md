# Notion 테스트 시나리오 DB 스키마

"Test Scenario" 데이터베이스의 실제 속성 정의.

- **DB 위치**: `<your-notion-parent-page>` > `<your-db-name>`
- **DB URL**: `<your-notion-db-url>`
- **data_source_id**: `<your-data-source-id>`

## 속성 (Properties)

| 속성명 | 타입 | 필수 | 설명 | 예시 |
|--------|------|------|------|------|
| 시나리오 ID | Title | ✅ | 시나리오 구분 고유번호. 스토리보드 화면 ID 사용 가능 | `LGN-001`, `SGN-001` |
| 시나리오명 | Text | ✅ | 무엇을 검증하는 시나리오인지 간단히 설명 | `유효한 이메일/비밀번호로 로그인 성공` |
| 사전조건 | Text | | 테스트 실행 전 필요한 환경, 데이터, 계정 상태 등 | `회원가입 완료된 계정이 존재` |
| 테스트 절차 | Text | ✅ | 사용자가 실제로 수행하는 구체적 단계 | `1. /login 접속\n2. 이메일 입력\n3. 비밀번호 입력\n4. 로그인 버튼 클릭` |
| 기대결과 | Text | ✅ | 단계 수행 시 시스템이 보여야 하는 결과 | `/dashboard로 리다이렉트, 사용자 이름 표시` |
| 카테고리 | Select | ✅ | 기능 영역 분류 | `로그인`, `아이디 찾기`, `아이디 설정`, `비밀번호 설정` |
| ID | Auto Increment | 자동 | 자동 증가 고유번호 (시스템 관리) | `1`, `2`, `3` |
| 티켓 생성 | Button | - | 버그 티켓 생성 버튼 (자동, 입력 불필요) | - |

## 행 삽입 시 사용할 속성

`notion-create-pages` 도구로 행을 삽입할 때 다음 속성만 지정:

```json
{
  "시나리오 ID": "LGN-001",
  "시나리오명": "유효한 이메일/비밀번호로 로그인 성공",
  "사전조건": "회원가입 완료된 계정이 존재",
  "테스트 절차": "1. /login 페이지 접속\n2. 이메일 입력란에 test@example.com 입력\n3. 비밀번호 입력란에 ValidP@ss123 입력\n4. 로그인 버튼 클릭",
  "기대결과": "/dashboard 페이지로 리다이렉트됨\n상단 헤더에 사용자 이름 표시",
  "카테고리": "로그인"
}
```

**주의사항:**
- `ID`와 `티켓 생성`은 시스템 관리 속성이므로 입력하지 않음
- `카테고리`는 기존 옵션 중 선택. 새 카테고리 필요 시 Notion에서 옵션 추가 후 사용

## 시나리오 ID 명명 규칙

카테고리별 접두사 + 순번:

| 카테고리 | 접두사 | 예시 |
|----------|--------|------|
| 로그인 | LGN | `LGN-001`, `LGN-002` |
| 아이디 찾기 | FID | `FID-001`, `FID-002` |
| 아이디 설정 | SID | `SID-001`, `SID-002` |
| 비밀번호 설정 | SPW | `SPW-001`, `SPW-002` |
| (새 카테고리 추가 시) | 영문 약어 3자 | `DSB-001` (대시보드) |

## 카테고리 옵션 (현재)

| 카테고리 | 대상 기능 |
|----------|----------|
| 로그인 | 이메일/비밀번호 로그인, Google OAuth 로그인, 로그아웃 |
| 아이디 찾기 | 아이디(이메일) 조회, 인증 |
| 아이디 설정 | 아이디(이메일) 변경, 신규 설정 |
| 비밀번호 설정 | 비밀번호 변경, 재설정, 분실 시 복구 |

> 새 카테고리가 필요하면 (대시보드, 프로젝트 등) Notion DB에서 Select 옵션을 먼저 추가한 뒤 사용.

## 테스트 절차 작성 규칙

- 번호 목록으로 작성 (`1. `, `2. `, ...)
- 각 단계는 하나의 사용자 동작 또는 검증 포인트
- 동작 단계: 사용자가 수행하는 액션 (접속, 클릭, 입력 등)
- 검증 단계: 결과를 확인하는 단계 (표시됨, 이동됨 등)
- 구체적인 UI 요소명 사용 (예: "로그인 버튼", "이메일 입력란")

### 예시

```
1. /login 페이지로 이동
2. 이메일 입력란에 "test@example.com" 입력
3. 비밀번호 입력란에 "ValidP@ss123" 입력
4. "로그인" 버튼 클릭
5. /dashboard 페이지로 리다이렉트되는지 확인
6. 상단 헤더에 사용자 이름이 표시되는지 확인
```
